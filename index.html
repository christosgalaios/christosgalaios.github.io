Sprint Skill — QA/Dev Communication

              
Built a custom sprint management skill where the Manager writes task specs to a shared sprint file and the Agent reads, implements, and acknowledges — all asynchronously. The Manager dispatches code-reviewer subagents after each commit, writes VERIFIED or feedback, and immediately queues the next task. No human in the loop. The sprint file is the single source of truth.

            

            

              
Autonomous Night Shift

              
When I say "night shift", the agents enter continuous operation. They never stop. When a task is done, they find the next one. When the backlog is empty, they proactively hunt for bugs, UX issues, missing validation, test coverage gaps. They commit working pieces early and often. Only stop when context runs out — then log what to do next so the next agent picks up seamlessly.

            

            

              
Learned-From-Mistakes System

              
Cross-project rules encoded from real failures: "always rebase before pushing", "workflow changes must land on the target branch", "never pass raw LLM responses to frontend", "never add auto-triggered API calls". Every mistake becomes a permanent rule. The agents don't repeat my mistakes — they've inherited 5 years of operational knowledge.

            

            

              
MCP Server Integration

              
SocialiseHub exposes 55+ tools via Model Context Protocol over SSE. QA agents can query the database, trigger browser automation, capture screenshots, and validate UI state — all through structured tool calls. The agents don't just write code; they test and operate the application autonomously.
